From src/commands/create.ts — the JIRA worktree creation block to reuse (D-04, lines 17-38):
```typescript
const gitRoot = await getGitRoot();
const spinner = p.spinner();
spinner.start(`Fetching JIRA ticket ${slug}...`);
const issue = await fetchIssue(slug);
const summarySlug = toSlug(issue.summary);
const ticketSlug = summarySlug ? `${slug}-${summarySlug}` : slug;  // empty slug fallback
spinner.stop(`Ticket: ${issue.summary}`);
const worktreePath = path.resolve(gitRoot, '..', ticketSlug);
const branch = `${type}/${ticketSlug}`;
const spinner2 = p.spinner();
spinner2.start(`Creating worktree ${ticketSlug}...`);
await gitWorktreeAdd(worktreePath, branch);
spinner2.stop(`Worktree created at ${worktreePath}`);
p.outro(`Branch: ${branch}`);
```

NOTE: In pick.ts, the ticket summary is already available from the selected issue object — no second fetchIssue() call needed. The ticketSlug is computed from the selected issue.summary directly (same logic, but using already-fetched data).

    - p.select() uses maxItems: 10 for scrollable list
    - After select: if p.isCancel(selected) → p.cancel('Cancelled.'), process.exit(0)
    - p.text() for branch type with message 'Branch type', placeholder 'feature', validate returns 'Branch type cannot be empty' when empty — per D-02
    - After text: if p.isCancel(typeResult) → p.cancel('Cancelled.'), process.exit(0)
    - Worktree creation uses the selected issue data directly (no second fetchIssue() call):
      - ticketSlug = summarySlug ? `${issue.key}-${summarySlug}` : issue.key  (same empty-slug fallback as create.ts)
      - worktreePath = path.resolve(gitRoot, '..', ticketSlug)
      - branch = `${type}/${ticketSlug}`
    - Worktree creation spinner: start 'Creating worktree {ticketSlug}...', stop 'Worktree created at {worktreePath}'
    - p.outro(`Branch: ${branch}`) on success
    - catch block: p.cancel(err.message), process.exit(1) for any git errors

    === pick.test.ts behaviors (all WT-05 test cases) ===
    - SUCCESS: correct ticket selected → fetchAssignedIssues called, p.select shown, p.text shown, gitWorktreeAdd called with correct path and branch
    - EMPTY STATE: issues.length === 0 → p.outro called with 'No assigned open tickets found.', gitWorktreeAdd NOT called
    - CANCEL ON SELECT: p.isCancel returns true for select result → p.cancel('Cancelled.') called, process.exit(0), gitWorktreeAdd NOT called
    - CANCEL ON TYPE: p.isCancel returns false for select, true for text result → p.cancel('Cancelled.') called, process.exit(0), gitWorktreeAdd NOT called
    - JIRA ERROR: fetchAssignedIssues rejects → spinner.stop('Failed to load tickets'), p.cancel called, process.exit(1)
    - SPINNER SHOWN: spinner.start called before fetchAssignedIssues resolves, spinner.stop called after
    - EMPTY SUMMARY FALLBACK: selected issue.summary slugifies to '' → ticketSlug falls back to issue.key alone
