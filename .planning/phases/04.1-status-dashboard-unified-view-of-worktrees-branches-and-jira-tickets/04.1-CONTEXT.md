# Phase 4.1: Status Dashboard - Context

**Gathered:** 2026-04-06
**Status:** Ready for planning

<domain>
## Phase Boundary

New `treeji status` command — unified dashboard showing worktrees, local branches, and JIRA tickets with their relationships. Shows all possible combinations of these three data sources, grouped by connection state.

</domain>

<decisions>
## Implementation Decisions

### Data sources
- **D-01:** Local branches: all except main/master. Show main/master ONLY if it has uncommitted changes or unpushed commits.
- **D-02:** JIRA tickets: default = assigned + open (`statusCategory != Done`). `--all` flag shows all assigned including closed.
- **D-03:** Worktrees: all from `git worktree list --porcelain` (existing `parseWorktreeList`).

### Combination coverage (all 7 states)
- **D-04:** Display ALL combinations — nothing hidden:
  1. JIRA + Branch + Worktree — fully connected, active work
  2. JIRA + Branch, no Worktree — branch exists but worktree removed
  3. JIRA only, no Branch, no Worktree — ticket assigned but work not started
  4. Branch + Worktree, no JIRA — local work without ticket (manual create)
  5. Branch only, no Worktree, no JIRA — orphaned branch
  6. Worktree only, no Branch match, no JIRA — detached HEAD or edge case
  7. JIRA + Worktree, no Branch match — rare mismatch

### Output format
- **D-05:** Group by connection state — sections like:
  - `Active work` (JIRA + branch + worktree)
  - `Branch without worktree` (JIRA + branch, no worktree)
  - `Not started` (JIRA only)
  - `Local only` (branch/worktree without JIRA)
  - `Orphaned` (branch only, no worktree, no JIRA)
  Section names and exact grouping at Claude's discretion.
- **D-06:** Default = compact format: `COREO-160 (In Progress) → feature/COREO-160-slug → ✓ worktree`
- **D-07:** `--full` flag = detailed format with all columns like `list` (name, status, branch, age, JIRA link)
- **D-08:** Empty sections are hidden (don't show "Not started" header if no tickets match)

### Matching logic
- **D-09:** Match branch ↔ JIRA by extracting ticket key from branch name via existing `extractTicketKey()` regex.
- **D-10:** Match worktree ↔ branch: worktree has a branch field in porcelain output.

### Claude's Discretion
- Exact section headers and emoji/symbols
- How to show the compact vs full detail per section
- Whether to add a summary line at top (e.g., "3 active, 2 not started, 1 orphaned")
- Color scheme for sections
- How to handle branches that don't match ticket key format (treat as "no JIRA")
- New git helper for listing all local branches (`git branch --list`)

</decisions>
