State definitions (D-04):
```
1: JIRA + Branch + Worktree     — Active work
2: JIRA + Branch, no Worktree   — Branch without worktree
3: JIRA only                    — Not started
4: Branch + Worktree, no JIRA   — Local only (with worktree)
5: Branch only, no JIRA         — Orphaned branch
6: Worktree only                — Detached or no branch match
7: JIRA + Worktree, no Branch   — Rare mismatch (branch deleted but worktree remains)
```

Create src/commands/status.ts following the architecture from RESEARCH.md:

**File structure:**

1. Imports — git helpers, jira helpers, colorStatus from list.ts, config, chalk, commander
2. Local extractTicketKey function (copy from list.ts — it is not exported there)
3. StatusEntry interface (state 1-7, jira, branch, worktree fields)
4. classify() function — pure function taking (jiraIssues, branches, worktrees) → StatusEntry[]
5. renderCompactRow() — one-liner format (D-06)
6. renderFullRow() — table row format (D-07); only called when entry.worktree !== null for git status calls
7. renderSection() — prints bold header, maps entries through renderRow, blank line after; skips if empty (D-08)
8. registerStatusCommand() — exported function wiring Commander action

**classify() algorithm** (follow RESEARCH.md Pattern 4 exactly):
```
Build three Maps:
  jiraByKey: Map<ticketKey, JiraIssue>
  worktreeByBranch: Map<branchName, WorktreeInfo>
  branchByTicketKey: Map<ticketKey, branchName>  (from filteredBranches via extractTicketKey)

Track visited sets to prevent double-counting:
  visitedJiraKeys: Set<string>
  visitedBranches: Set<string>
  visitedWorktreePaths: Set<string>

Pass 1 — Walk jiraIssues:
  For each issue:
    branch = branchByTicketKey.get(issue.key) ?? null
    worktree = branch ? worktreeByBranch.get(branch) ?? null : null
    
    If no branch found, check if any worktree has extractTicketKey(wt.branch) === issue.key
    AND that worktree.branch is NOT in filteredBranches → state 7
    
    Otherwise:
      has branch AND worktree → state 1
      has branch, no worktree → state 2
      no branch, no worktree → state 3
    
    Mark jiraKey, branch, worktree path as visited

Pass 2 — Walk filteredBranches not yet visited:
  worktree = worktreeByBranch.get(branch) ?? null
  has worktree → state 4
  no worktree → state 5
  Mark branch and worktree path as visited

Pass 3 — Walk worktrees not yet visited (by path):
  state 6
```

**Main/master filter** (D-01 — follow RESEARCH.md Pattern 2):
- Find mainWorktree = worktrees.find(wt => wt.isMain)
- If mainWorktree exists, run gitStatusPorcelain + gitAheadBehind in parallel on mainWorktree.path
- includeMain = status non-empty OR ahead > 0
- Filter: branches = branches.filter(b => !MAIN_BRANCHES.has(b) || includeMain)
- MAIN_BRANCHES = new Set(['main', 'master'])

**Section rendering** (D-05 — suggested section names at discretion):
```
Active work     → state 1 entries
Branch only     → state 2 entries (JIRA + branch, no worktree)
Not started     → state 3 entries (JIRA only)
Local work      → state 4 entries (branch + worktree, no JIRA)
Orphaned        → state 5 entries (branch only, no JIRA)
Detached        → state 6 entries (worktree only)
Mismatched      → state 7 entries (JIRA + worktree, no branch)
```
Use chalk.bold for section headers. Blank line between sections.

**Compact format** (D-06 — one line per entry):
- Build array of parts, join with chalk.dim(' → ')
- JIRA part: OSC 8 hyperlink key + colorStatus in parens (use jiraCell pattern from list.ts)
- Branch part: chalk.cyan(branch)
- Worktree part: chalk.green('✓ worktree')
- Example: `COREO-160 (In Progress) → feature/COREO-160-slug → ✓ worktree`
- Prefix with 2 spaces: `console.log('  ' + parts.join(chalk.dim(' → ')))`

**Full format** (D-07 — column table):
- Columns: JIRA | branch | status | age | path
- For entries with no worktree (states 2, 3, 5): status='—', age='—'
- Only call gitStatusPorcelain/gitAheadBehind/gitLastCommitRelativeDate when entry.worktree !== null
- Use padEnd for alignment; compute max widths before rendering (same pattern as list.ts)
