Create `src/lib/jira.ts` implementing the three components:

**1. createJiraClient() (internal, not exported)**
```typescript
import { Version3Client } from 'jira.js';
import { loadConfig } from './config.js';
import { getToken } from './keychain.js';

function createJiraClient(): Version3Client {
  const config = loadConfig();
  if (!config.host || !config.email) {
    throw new Error('JIRA not configured. Run `mafcli configure` first.');
  }
  const token = getToken(config.email);
  if (!token) {
    throw new Error('JIRA API token not found. Run `mafcli configure` first.');
  }
  return new Version3Client({
    host: config.host,
    authentication: { basic: { email: config.email, apiToken: token } },
  });
}
```

**2. withRetry<T>() (internal, not exported)**
- maxAttempts = 3, initial delay = 400ms, doubles each retry
- Detect 429 by checking error message for '429' OR 'Too Many Requests' (case-insensitive)
- Non-429 errors throw immediately (no retry)
- After maxAttempts exhausted, rethrow last error
- See RESEARCH.md lines 129-149 for exact implementation

**3. fetchIssue(ticketKey: string) (exported)**
- Calls createJiraClient(), then withRetry(() => client.issues.getIssue({ issueIdOrKey: ticketKey, fields: ['summary', 'status', 'issuetype'] }))
- Returns { key: ticketKey, summary: string, statusName: string }
- Cast fields: `(issue.fields?.summary as string | undefined) ?? ticketKey`
- Cast status: `(issue.fields?.status as { name?: string } | undefined)?.name ?? ''`

**4. fetchIssueStatuses(ticketKeys: string[]) (exported)**
- Guard: if ticketKeys.length === 0, return new Map() immediately (no API call)
- JQL: `issue in (${ticketKeys.join(', ')})`
- Calls searchForIssuesUsingJqlEnhancedSearchPost({ jql, fields: ['status'], maxResults: ticketKeys.length })
- Returns Map<string, string> where key = issue.key, value = status name
- Cast status same way as fetchIssue
