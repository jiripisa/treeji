## What Was Built

### `gitBranchMergedInto(branch, target)` — `src/lib/git.ts`

New exported function using `git merge-base --is-ancestor`. Wrapped in try/catch: resolves to `true` on exit code 0 (merged), `false` on any error including exit code 1 (not merged) and unexpected errors. Safe-default behavior: when the git command fails for unexpected reasons, deletion is blocked pending user confirmation.

### Merge-check in interactive picker — `src/commands/remove.ts`

After `p.select()` and `p.isCancel()` check, the flow now:
1. Calls `gitBranchMergedInto(branch, 'main')`
2. If merged: proceeds to deletion immediately (no prompt)
3. If not merged: shows `p.confirm()` with `initialValue: false` (safe default is "no")
4. User accepts: proceeds to deletion
5. User declines or cancels: `p.cancel('Aborted.')` + `process.exit(1)`

The existing `gitDeleteBranch(branch, false)` in the interactive block changed to `gitDeleteBranch(branch, true)` — force=true is safe because the branch is already confirmed to be on remote AND either merged or explicitly approved by the user.
