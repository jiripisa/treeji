  <files>src/lib/branch-type.ts, src/commands/pick.ts, src/commands/pick.test.ts</files>
  <behavior>
    - promptBranchType() returns string for standard types (feature, fix, etc.)
    - promptBranchType() returns empty string for "none" selection
    - promptBranchType() prompts p.text() when "custom" selected, returns user input
    - promptBranchType() calls process.exit(0) when user cancels select
    - promptBranchType() calls process.exit(0) when user cancels custom text input
    - BRANCH_TYPE_OPTIONS has 9 entries: feature, fix, refactor, docs, style, test, chore, custom, none
    - pick.ts: branch name omits type prefix when promptBranchType returns empty string
  </behavior>
  <action>
    1. Create `src/lib/branch-type.ts`:
       - Export `BRANCH_TYPE_OPTIONS` array with these entries (each has `value` and `hint`):
         - `feature` — "new feature for the user, not a new feature for a build script"
         - `fix` — "bug fix for the user, not a fix to a build script"
         - `refactor` — "refactoring production code, e.g., renaming a variable"
         - `docs` — "changes to the documentation"
         - `style` — "formatting, missing semicolons, etc.; no production code change"
         - `test` — "adding missing tests, refactoring tests; no production code change"
         - `chore` — "updating grunt tasks etc.; no production code change"
         - `custom` — "type your own branch type"
         - `none` — "no branch type prefix"
       - Export `async function promptBranchType(): Promise<string>` that:
         - Calls `p.select()` with message "Branch type" and the options above
         - If cancelled: `p.cancel('Cancelled.')` + `process.exit(0)`
         - If "custom": calls `p.text({ message: 'Custom branch type', validate: v => !v ? 'Cannot be empty' : undefined })`
           - If text cancelled: `p.cancel('Cancelled.')` + `process.exit(0)`
           - Returns the custom text string
         - If "none": returns empty string `""`
         - Otherwise: returns the selected value string (e.g., "feature")

    2. Update `src/commands/pick.ts`:
       - Replace `import` — add `promptBranchType` from `../lib/branch-type.js`
       - Replace the `p.text()` block (lines 58-70) with: `const type = await promptBranchType();`
       - Update branch name construction: `const branch = type ? `${type}/${ticketSlug}` : ticketSlug;`

    3. Update `src/commands/pick.test.ts`:
       - Add mock for `../lib/branch-type.js` with `mockPromptBranchType` returning `'feature'` by default
       - Update existing tests that set up `mockText` for branch type to use `mockPromptBranchType` instead
       - Add test: when promptBranchType returns empty string, branch name is just the ticketSlug (no prefix)
  </action>
