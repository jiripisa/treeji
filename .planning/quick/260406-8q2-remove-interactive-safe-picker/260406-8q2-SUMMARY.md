## What Was Built

### gitBranchExistsOnRemote (git.ts)

New exported async helper that runs `git branch -r --list "origin/{branch}"` and returns `true` if stdout is non-empty after trimming. Catches all execa errors and returns `false` — callers never need error handling.

### Interactive picker mode (remove.ts)

`treeji remove` now accepts an optional `[name]` argument:

- **With name:** all existing behavior unchanged (named remove flow)
- **Without name:** interactive picker flow
  1. List all worktrees via `gitWorktreeList` + `parseWorktreeList`
  2. Filter: exclude main worktree, exclude detached HEAD (branch === null)
  3. For each candidate: check `gitStatusPorcelain` (must be clean) and `gitBranchExistsOnRemote` (must exist on remote)
  4. If zero safe worktrees: `p.outro('No worktrees can be safely removed')` + `process.exit(0)`
  5. Otherwise: `p.select()` with label=basename, hint=branch
  6. If cancelled: `p.cancel('Aborted.')` + `process.exit(1)`
  7. If selected: `gitWorktreeRemove` → `gitDeleteBranch` → `gitWorktreePrune` (no --force, same pattern as named mode)
