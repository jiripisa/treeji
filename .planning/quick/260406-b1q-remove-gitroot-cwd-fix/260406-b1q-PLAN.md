<objective>
Fix `treeji remove` when run from inside the worktree being removed.

After `git worktree remove` deletes the directory, the shell CWD no longer exists. Any subsequent git command (`git branch -d`, `git worktree prune`) that relies on CWD to find the repo fails with a "not a git repository" error.

Purpose: Ensure all git commands in the remove flow execute against the main repo root, not the soon-to-be-deleted CWD.
Output: Modified git.ts (3 functions), remove.ts (getGitRoot call + threading), remove.test.ts (updated assertions).
</objective>
