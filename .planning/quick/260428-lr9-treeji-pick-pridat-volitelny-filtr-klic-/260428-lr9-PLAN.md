<objective>
Add an optional positional argument to `treeji pick` so the user can narrow the ticket list. `treeji pick` keeps current behaviour. `treeji pick ABC` shows only tickets where `ABC` (case-insensitive substring) appears in the JIRA `key` or `summary`. The argument also widens scope to include closed tickets so the user can find an old ticket by name.

Purpose: Reduce friction when the user knows partial ticket info and the assigned list is long or contains the closed item they want to revisit.
Output: Updated `pick` command + tests. No other commands or libraries touched.
